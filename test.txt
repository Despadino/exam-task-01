1. Чтобы перейти в директорию, используйте команду "cd", а затем укажите путь к нужной директории. Например, "cd Documents" для перехода в папку "Documents".
2. "~" (тильда) обозначает домашнюю директорию пользователя. "." (точка) обозначает текущую директорию, а ".." (две точки) обозначает родительскую директорию.
3. "rm" - это команда в UNIX-подобных системах для удаления файла или папки.
4. "cd" - это команда для изменения текущей рабочей директории. Она позволяет перейти в другую директорию.
5. "cp" - это команда для копирования файла или папки.
6. "mv" - это команда для перемещения файла или папки. Она также может быть использована для переименования файлов или папок.
7. Чтобы переименовать папку или файл, используйте команду "mv" с двумя аргументами: первый аргумент - текущее имя папки/файла, второй аргумент - новое имя папки/файла. Например, "mv oldfolder newfolder" для переименования папки.
8. Чтобы переместить файл или папку, используйте команду "mv" с двумя аргументами: первый аргумент - текущий путь к файлу/папке, второй аргумент - новый путь к файлу/папке. Например, "mv file.txt Documents/" для перемещения файла в папку "Documents".
9. Флаг "-R" используется с командами "cp" и "mv" и указывает на рекурсивное копирование или перемещение. Он позволяет копировать или перемещать содержимое папки вместе с поддиректориями и их содержимым.
10. Флаг "-r" используется с командой "rm" и указывает на рекурсивное удаление, то есть удаление папки и ее содержимого.
11. Git - это система контроля версий, которая позволяет управлять изменениями в файловой системе и сотрудничать с другими разработчиками.
12. "git init" - это команда создания нового репозитория Git. Она инициализирует пустой репозиторий в текущей директории.
13. Чтобы создать новую ветку в Git, используйте команду "git branch" с указанием имени ветки. Например, "git branch newbranch" для создания новой ветки с именем "newbranch".
14. Чтобы переключиться на другую ветку в Git, используйте команду "git checkout" с указанием имени ветки. Например, "git checkout newbranch" для переключения на ветку с именем "newbranch".
15. Слияние (merge) в Git - это процесс объединения изменений из одной ветки в другую. Обычно это делается для интеграции новых функций или исправлений ошибок.
16. "git rm" - это команда для удаления файла из репозитория Git. Она также удаляет файл из файловой системы.
17. "git revert" - это команда в Git, которая создает новый коммит, отменяющий изменения, внесенные в предыдущий коммит. Таким образом, она позволяет откатить изменения и вернуть репозиторий к предыдущему состоянию.
